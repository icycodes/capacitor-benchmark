### 1. `android/app/src/main/AndroidManifest.xml` тАФ App-Link intent filter added

A second `<intent-filter>` was inserted inside `.MainActivity`, alongside the original `MAIN`/`LAUNCHER` filter:

```xml
<intent-filter android:autoVerify="true">
    <action android:name="android.intent.action.VIEW" />
    <category android:name="android.intent.category.DEFAULT" />
    <category android:name="android.intent.category.BROWSABLE" />
    <data
        android:scheme="https"
        android:host="myapp.example.com"
        android:pathPrefix="/.well-known" />
</intent-filter>
```

- `android:autoVerify="true"` promotes this from a plain deep link to a verified **App Link**, so Android skips the disambiguation dialog.
- The `https` scheme + `myapp.example.com` host combination matches the domain that serves `assetlinks.json`.
- `android:pathPrefix="/.well-known"` satisfies the requirement to cover URLs under `/.well-known/` on that host.

### 3. `src/deeplink.ts` тАФ TypeScript deep-link handler

```ts
import { App } from '@capacitor/app';

export function registerDeepLinkHandler(): void {
  App.addListener('appUrlOpen', (event) => {
    const url = new URL(event.url);
    const pathname = url.pathname;
    window.history.replaceState(null, '', pathname);
    window.dispatchEvent(new PopStateEvent('popstate', { state: null }));
  });
}
```

- Named import of `App` from `@capacitor/app`.
- Subscribes to `appUrlOpen`; parses the full URL with `new URL()` to extract `pathname`.
- Updates the browser's history entry with `window.history.replaceState` (no full reload) and fires a synthetic `popstate` event so any history-based SPA router automatically reacts to the route change.
- Exported named function тАФ call `registerDeepLinkHandler()` once from `src/index.ts` (or wherever your app bootstraps).
