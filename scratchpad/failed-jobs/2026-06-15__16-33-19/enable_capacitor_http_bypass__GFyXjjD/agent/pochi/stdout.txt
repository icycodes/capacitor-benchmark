Imports `CapacitorHttp` (and its `HttpOptions` type) directly from `@capacitor/core` — no `fetch`, `axios`, `node-fetch`, `http`, or `https` modules are used.

**Argument parsing** uses a plain `process.argv` loop (no extra packages):

| Flag | Required | Description |
|---|---|---|
| `--method` | ✅ | `GET` or `POST` |
| `--url` | ✅ | Absolute URL |
| `--body` | optional | JSON string sent as request body |
| `--header` | optional | `Name:Value` extra header forwarded verbatim |

**Stdout format (exactly two lines):**
```
Status: <numeric_status_code>
<json-serialized response body>
```

**Usage examples verified live:**

```bash
# GET
npx tsx scripts/http-cli.ts --method GET --url https://httpbin.org/get
# → Status: 200
# → {"args":{},"headers":{...},...}

# POST with body + custom header (e.g. run-id forwarding)
npx tsx scripts/http-cli.ts \
  --method POST \
  --url https://httpbin.org/post \
  --body '{"hello":"world"}' \
  --header "X-Run-Id:test-42"
# → Status: 200
# → {"args":{},"data":"{\"hello\":\"world\"}","headers":{"X-Run-Id":"test-42",...},...}
```

The `--header` flag is forwarded faithfully to the remote endpoint, so `ZEALT_RUN_ID` / run-id values injected by the verifier via `--header` will reach the server as-is.
